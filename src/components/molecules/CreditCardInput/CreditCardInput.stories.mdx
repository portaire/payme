# Credit Card Input

<br />
<br />

## Approach

I've taken a more composable approach here, where the CreditCardInput is a wrapper around the individual components that make up the input. This allows for more flexibility in the future, and also allows for the CreditCardInput to be used in a more flexible way due to the lack of props that have to be passed to a single component.

<br />

This component is also polymorphic, meaning that it can be used as a div, or a
form, or any html element depending on the use case.

<b>
  This component is designed to be used in together with useCreditCardInput in
  @hooks
</b>
